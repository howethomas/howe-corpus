# Talks

Conference talks, podcasts, BoF sessions, keynotes, and interviews — each
as a structured vCon.

Entries are indexed in [`../../corpus.yaml`](../../corpus.yaml) under
`items:` with `type: talk` or `type: podcast`. When you lift an index
entry into a full vCon, save it here as `<id>.vcon.json`, reusing the
same `id`.

See [`../../schema/vcon.md`](../../schema/vcon.md) for the expected shape.
For talks, `parties` should include speaker(s) and (for podcasts) the
host/interviewer; `attachments` should link to the video/audio;
`dialog` should include a text summary or transcript.
